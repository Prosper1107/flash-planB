Demarrage rapide

Installation en 3 etapes

Aucune dependance externe. Un seul fichier JavaScript suffit.

1

Ajouter le script dans votre page HTML

Copiez cette ligne avant la fermeture de votre balise </body>

<script src="flash-sdk.js"></script>
2

Initialiser le SDK avec votre tag Flash

Remplacez votre-tag par le tag de votre compte Flash

FlashPay.init({ merchantTag: 'votre-tag', onSuccess: function(payment) { console.log('Paiement recu', payment); // Declenchez votre logique metier ici }, onError: function(err) { console.error('Erreur paiement', err); } });
3

Inserer un bouton de paiement

Ajoutez un conteneur vide dans votre HTML, le SDK cree le bouton automatiquement

<!-- Dans votre HTML --> <div id="pay-btn"></div> <!-- Dans votre JavaScript --> FlashPay.button('#pay-btn', { amount: 5000, currency: 'XOF', description: 'Commande #123' });

API avancee

Checkout programmatique

Ouvrez le modal depuis n'importe quel evenement de votre application.

// Ouvrir le modal et conserver une reference const modal = FlashPay.checkout({ amount: 15000, description: 'Abonnement Premium', onSuccess: function(payment) { console.log('Paiement recu', payment.amount, payment.sats); // payment.amount → montant XOF // payment.sats → satoshis correspondants // payment.lightningAddress → adresse Lightning du marchand // payment.timestamp → ISO 8601 // payment.status → "simulated" | "completed" }, onClose: function() { console.log('Modal ferme par l\'utilisateur'); } }); // Fermer le modal depuis votre code modal.close();
Utilitaires

Fonctions de conversion et formatage

Outils inclus dans le SDK pour faciliter vos calculs.

// Obtenir la Lightning Address du marchand FlashPay.getLightningAddress(); // → "votre-tag@bitcoinflash.xyz" // Convertir XOF en satoshis FlashPay.xofToSats(5000); // → 13157 (avec ratePerSat = 0.38) // Convertir satoshis en XOF FlashPay.satsToXof(10000); // → 3800 // Formater un montant XOF en texte lisible FlashPay.formatXof(5000); // → "5 000 XOF" // Formater des satoshis en texte lisible FlashPay.formatSats(1500000); // → "1.50M sats"
Appels API directs

Methodes API Flash

Ces methodes communiquent directement avec l'API Flash. Elles necessitent un token JWT ou un X-Staging-User-Id dans la configuration.

// Configuration avec authentification FlashPay.init({ merchantTag: 'votre-tag', token: 'votre_jwt_token', // OU en staging : stagingUserId: 'votre_staging_user_id', }); // Creer une transaction de vente (sats → Mobile Money) FlashPay.createTransaction({ type: 'SELL_BITCOIN', amount: 13157, // satoshis number: '+22997000000', // numero avec indicatif provider: 'MTN_OPEN', // voir tableau operateurs }).then(function(data) { console.log('Transaction', data); }); // Recuperer les transactions FlashPay.getTransactions().then(function(data) { console.log(data.transactions); }); // Recuperer la balance FlashPay.getBalance().then(function(data) { console.log(data); }); // Recuperer le taux de change depuis l'API FlashPay.getRate().then(function(data) { FlashPay.setRate(data.rate_per_sat); });
Reference

Configuration FlashPay.init()

Parametres acceptes par la methode d'initialisation.

Parametre Type Defaut Description
merchantTag Requis string - Tag Flash du marchand. Genere l'adresse tag@bitcoinflash.xyz
apiBase Optionnel string staging.bitcoinflash.xyz URL de base de l'API Flash
token Optionnel string - JWT Bearer token pour les requetes authentifiees
stagingUserId Optionnel string - Header X-Staging-User-Id pour l'environnement de staging
ratePerSat Optionnel number 0.38 Taux XOF par satoshi utilise pour les conversions
onSuccess Optionnel function - Callback (payment) => void appele apres paiement confirme
onError Optionnel string - Callback (error) => void appele en cas d'echec
onClose Optionnel function - Callback () => void appele a la fermeture du modal
